## Chapter Overview
- A queue is a linear data structure that follows the FIFO (First In, First Out) principle.
    - **Insertion** : happens at the **rear**
    - **Deletion** : happens at the **front**
- It works like a line at a grocery store: the first person in line is the first one served.
- Queues can be implemented using:
    - Arrays
    - Linked lists

## Key Concepts

### Linear Queues
- array-based implementation
- Problem: wasted space after deletions (front moves forward, but freed space is not reused).

### Circular Queues
- The last index connects back to index 0.
- Solves memory wastage problem of linear queues.
- Full condition (array implementation):
    - (rear + 1) % MAX == front
 
### Linked Queues
- Implemented using linked lists.
- No fixed size.
- Storage requirement: O(n)
- Operations (enqueue/dequeue): O(1)

### Deque (Double Ended Queue)
- Insertion and deletion allowed at both ends.
- Cannot insert/delete from the middle.
- Implemented using:
    - Circular array
    - Circular doubly linked list
- Types:
    - **Input-restricted deque** : Insert at one end, delete at both.
    - **Output-restricted deque** : Delete at one end, insert at both.

### Priority Queues
- Each element has a priority.
- Higher priority elements are processed first.
- If two elements have same priority -> FCFS order.
- Node contains:
    - Data
    - Priority
    - Next pointer

### Applications of Queues
- CPU scheduling
- Printer spooling
- I/O buffers
- Interrupt handling
- Playlists
- Josephus Problem

## Review Questions

### 1. What is a priority queue? Give its applications.
- A priority queues is a queue where each element has a priority. The value of the priority influences the time of the output of the element, rather than just the arrival time. This is used in CPU scheduling in operating systems

### 2. Explain the concept of a circular queue. How is it better than a linear queue?
- In a linear queue, once the rear reaches the end of the array, no further insertions are possible even if there is unused space at the beginning due to deletions. A circular queue eliminates this inefficiency by wrapping around using modular arithmetic.

### 3. Why do we use multiple queues?
- Multiple queues are used when we need to manage different categories of data or processes separately within the same memory space. For example, an operating system may maintain different queues for different priority levels of tasks. Using multiple queues allows better organization and scheduling flexibility.

### 4 & 5. Queue Operation Sequences
- When elements A, B, C, D, E, and F are added to an empty queue, they are inserted at the rear, forming the sequence A B C D E F. After deleting two elements, A and B are removed because deletion occurs at the front, leaving C D E F. Adding G and H appends them at the rear, forming C D E F G H. Deleting four elements removes C, D, E, and F, leaving G H. Finally, adding I results in G H I. The key principle demonstrated here is that insertion always happens at the rear and deletion always happens at the front, preserving FIFO order.

### 6. Deque Operations
- A deque allows insertion and deletion at both ends. Starting with A B C D E, adding F to the left places it at the front, forming F A B C D E. Adding G and H to the right appends them to the rear. Deleting two elements from the left removes the two front elements. After additional insertions and deletions, the structure changes accordingly.
